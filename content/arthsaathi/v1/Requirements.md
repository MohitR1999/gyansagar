Before we start thinking about 'how' to build, we must think of 'what' to build. 
### So, 'what' to build?
Well, I want to build a financial assistant that will help me keep track of the expenses that I do in a day, in a week, in a month, in a year, and so on. Based on this data, I'd like to gather insights on where do I spend the most, and is it possible to reduce unnecessary expenses? This, I'd say, is our problem statement
### Okay, now what 'exactly' you want to do?
Well, there are a couple of categories of 'requirements' that my system must be able to fulfil, and those are the **Functional** and **Non-Functional** requirements.
### What do you mean? Explain like I'm five
- Functional requirements, as the name suggests, deal with the overall ***functions*** of the system. These outline exactly what the system should do. They stress upon the ***features*** of the system.
- Non functional requirements, they deal with ***how*** the system should operate. They stress upon the usability, performance, reliability, and user experience. They ensure that the system is *efficient*, *secure* and *maintainable* over time.
If you want to learn more, there's a great article on GeeksForGeeks [here](https://www.geeksforgeeks.org/software-engineering/functional-vs-non-functional-requirements/)
### Got it, so tell me these requirements exactly now
Here you go:

- **Functional Requirements**
    - Authentication
        -  I should be able to register and create an account
        - I should be able to log in with my username and password
    - Categorization
        - I should be able to specify different categories for expenses and income
    - Cash flow handling
        - Expense monitoring
            - I should be able to create/edit/delete an expense with date and time
            - I should be able to categorize an expense
        - Income monitoring
            - I should be able to create/edit/delete an income
            - I should be able to categorize an income
    - Report generation
        - I should be able to generate daily report
        - I should be able to generate monthly report
        - I should be able to generate yearly report
- **Non Functional Requirements**
    - Beautiful UI: The user interface should be intuitive and pleasant to use
    - Dark mode support: Eye friendly
    - High uptime: 99.999% availability

Please keep in mind that this list might not be the perfect example of the concepts I just talked about, and I'll continue to improve upon this as my knowledge and understanding grows.
